üìñ Cybersecurity Glossary

Common terms and definitions to help you understand cybersecurity concepts.

A-E

  • Authentication: The process of verifying a user's identity
  • Backup: A copy of data stored separately for recovery purposes
  • Botnet: A network of infected computers controlled by hackers
  • Breach: Unauthorized access to data or systems
  • Cloud Storage: Data stored on remote servers accessed via the internet
  • Cookie: Small file stored by websites to remember user information
  • Cyberbullying: Harassment or intimidation using digital devices
  • Data Encryption: Converting data into coded format to prevent unauthorized access
  • DDoS (Distributed Denial of Service): Attack overwhelming a server with traffic
  • End-to-End Encryption: Data encrypted throughout entire transmission

F-M

  • Firewall: Security system that monitors and controls network traffic
  • HTTPS: Secure version of HTTP using encryption
  • IoT (Internet of Things): Network of connected devices
  • Keylogger: Software that records keystrokes to steal information
  • Malware: Malicious software designed to harm or exploit systems
  • Multi-Factor Authentication (MFA): Using multiple methods to verify identity

N-S

  • Patch: Software update that fixes security vulnerabilities
  • Phishing: Fraudulent emails attempting to steal sensitive information
  • Ransomware: Malware that encrypts data and demands payment
  • Social Engineering: Manipulating people to divulge confidential information
  • Spam: Unsolicited, often malicious, email messages
  • Spyware: Software that secretly monitors user activity
  • SSL/TLS: Protocols for secure internet communication

T-Z

  • Trojan: Malware disguised as legitimate software
  • Two-Factor Authentication (2FA): Security requiring two forms of verification
  • Virus: Malicious code that replicates and spreads to other files
  • VPN (Virtual Private Network): Encrypted connection for secure internet access
  • Whaling: Phishing attack targeting high-level executives
  • Zero-Day: Previously unknown vulnerability exploited by attackers

üåç Real-World Examples

Learn from actual cybersecurity incidents and how to avoid similar situations.

Example 1: The Password Reuse Incident

What Happened: A small business owner used the same password across multiple accounts. When a shopping website was breached, hackers obtained their password and used it to access their business email and bank account.

Impact: $15,000 stolen from business account, customer data compromised.

Lesson: Always use unique passwords for different accounts, especially for financial and business services.

Prevention: Use a password manager to generate and store unique passwords.

Example 2: The Phishing Email

What Happened: An employee received an email appearing to be from the CEO requesting an urgent wire transfer. The email looked legitimate but came from a spoofed address.

Impact: $50,000 transferred to fraudulent account before discovering the scam.

Lesson: Always verify unusual requests, especially those involving money, through a separate communication channel.

Prevention: Implement email authentication (DMARC), train employees to recognize phishing, establish verification procedures for financial transactions.

Example 3: The Unsecured Smart Camera

What Happened: A family installed a smart camera with the default password unchanged. Hackers accessed the camera feed and used it to monitor the home.

Impact: Privacy violation, potential security risk to family.

Lesson: Always change default passwords on IoT devices immediately after installation.

Prevention: Use strong, unique passwords for all smart devices, keep firmware updated, use a separate network for IoT devices.

Example 4: The Unpatched Software

What Happened: A business delayed installing security updates on their software. Ransomware exploited a known vulnerability, encrypting all company data.

Impact: 3 days of downtime, $30,000 in lost revenue, $10,000 to recover systems.

Lesson: Install security updates promptly‚Äîdelays create windows of vulnerability.

Prevention: Enable automatic updates, have a patch management policy, maintain offline backups.

Example 5: The Social Media Overshare

What Happened: A person posted about going on vacation with dates and location. Burglars monitored social media and robbed the home while they were away.

Impact: Property loss, emotional distress, increased insurance premiums.

Lesson: Don't announce when your home will be unoccupied. Share vacation photos after returning.

Prevention: Review privacy settings, limit post visibility, wait to share travel updates.

Example 6: The Public Wi-Fi Mistake

What Happened: Someone accessed their bank account on public Wi-Fi at a coffee shop. A hacker on the same network intercepted the login credentials.

Impact: Unauthorized access to bank account, fraudulent transactions.

Lesson: Never access sensitive accounts on public Wi-Fi without using a VPN.

Prevention: Use VPN on public networks, avoid accessing financial accounts in public places, use mobile data instead.
